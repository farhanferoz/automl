# Python instruction-level coding dataset dossier (R5a)

Groundwork dossier for FlexNN programme Task F11 (R5a). Scope: inventory public
**instruction-level** (question/instruction → code solution, not raw-code pretraining
corpora like The Stack itself) Python coding datasets, resolve license/size/provenance/dedup
per dataset, and recommend a Python-only corpus mix with a token estimate. Non-goals (per
brief): no purchases/vendor recommendations, no dataset downloads, no architecture content.

Every factual claim below carries an inline source URL that was fetched (WebSearch/WebFetch)
during this dossier's authoring run (2026-07-18). Anything not resolvable from a fetched
source this run is listed in **OPEN / unresolved**, not asserted as fact.

## 1. Dataset-by-dataset findings

### 1.1 CodeAlpaca-20k (seed anchor)

- **Size:** 20,022 rows ([sahil2801/CodeAlpaca-20k](https://huggingface.co/datasets/sahil2801/CodeAlpaca-20k)); the original author's GitHub repo states "20,000 instruction-following examples" and "each of the 20K instructions is unique" ([sahil280114/codealpaca](https://github.com/sahil280114/codealpaca)).
- **License — CONFLICTING SOURCES:** the HF mirror `sahil2801/CodeAlpaca-20k` is tagged **CC-BY-4.0** ([HF page](https://huggingface.co/datasets/sahil2801/CodeAlpaca-20k)); the original GitHub repository states **Apache-2.0** in its license badge/file ([GitHub](https://github.com/sahil280114/codealpaca)). These disagree — treat as unresolved until the canonical repo's LICENSE file is read directly (not just the badge), see OPEN list. Other HF mirrors add further variants (`flwrlabs/code-alpaca-20k` = Apache-2.0) ([search result](https://huggingface.co/datasets/flwrlabs/code-alpaca-20k)).
- **Provenance:** Self-Instruct method (Stanford Alpaca's approach) adapted to code — "modified prompt to focus on code generation/editing/optimization tasks," seed tasks restricted to code; **responses generated by `text-davinci-003`** (OpenAI), i.e. model-generated, not human-written ([GitHub README](https://github.com/sahil280114/codealpaca)).
- **Language coverage:** multi-language (Python, Java, C++, JavaScript, SQL, etc. observed in samples per the HF page) — **Python-only fraction not verified this run** (OPEN).
- **Dedup:** author claims instruction-level uniqueness ("each of the 20K instructions is unique") but no formal dedup algorithm is described ([GitHub](https://github.com/sahil280114/codealpaca)).
- **Contamination risk:** model-generated (`text-davinci-003`), no stated benchmark decontamination step.

### 1.2 Evol-CodeAlpaca-V1 (seed anchor)

- **Size:** 111,272 rows ([theblackcat102/evol-codealpaca-v1](https://huggingface.co/datasets/theblackcat102/evol-codealpaca-v1)).
- **License:** **Apache-2.0** ([HF page](https://huggingface.co/datasets/theblackcat102/evol-codealpaca-v1)).
- **Provenance:** Evol-Instruct methodology applied to `HuggingFaceH4/CodeAlpaca_20K`; "used the `gpt-4-0314` and `gpt-4-0613` models to augment and answer each response, with the bulk of generation handled by `gpt-4-0314`," across 10 augmentation strategies including a language-conversion (Chinese) strategy; a 2023 update filtered for "pure english instruction" and removed OAI self-references ([HF page](https://huggingface.co/datasets/theblackcat102/evol-codealpaca-v1)). Fully model-generated (GPT-4).
- **Language coverage:** inherits CodeAlpaca's multi-language spread (not stated as Python-filtered) — **Python-only fraction not verified this run** (OPEN).
- **Dedup:** the dataset card itself warns "the same questions can be found in these dataset[s] [e.g. teknium/OpenHermes-2.5], so be sure to deduplicate when training" — i.e. cross-dataset overlap is flagged by the authors, but **no in-dataset dedup step by the creators is described** ([HF page](https://huggingface.co/datasets/theblackcat102/evol-codealpaca-v1)).
- **Token stats:** card states median sequence length of 471 tokens ([HF page](https://huggingface.co/datasets/theblackcat102/evol-codealpaca-v1)) → 111,272 × 471 ≈ **52.4M tokens** (derived; median×count is an approximation, not a true sum).

### 1.3 Magicoder-OSS-Instruct-75K (seed anchor, arXiv:2312.02120)

- **Size:** 75,197 rows, 203 MB total file size ([ise-uiuc/Magicoder-OSS-Instruct-75K](https://huggingface.co/datasets/ise-uiuc/Magicoder-OSS-Instruct-75K)).
- **License:** dataset card tags **MIT**, but explicitly flags a caveat: "This is the OSS-Instruct dataset generated by `gpt-3.5-turbo-1106` ... Please pay attention to OpenAI's usage policy when adopting this dataset" ([HF page](https://huggingface.co/datasets/ise-uiuc/Magicoder-OSS-Instruct-75K)) — i.e. MIT applies to the data structure/text itself, but OpenAI's usage-policy terms are a separate overlay the card asks users to respect.
- **Provenance:** seeded from **starcoderdata**, "a filtered version of The Stack dataset that StarCoder is trained on, containing permissively licensed source code documents" — 80K seed snippets (40K Python + 5K each of C++, Java, TypeScript, Shell, C#, Rust, PHP, Swift); instructions/responses generated by **`gpt-3.5-turbo-1106`** ("due to its high cost-effectiveness") ([ar5iv paper](https://ar5iv.labs.arxiv.org/html/2312.02120)). Fully model-generated.
- **Language coverage:** paper's ablation study classifies the **final** 75K set as **~43K Python-only vs ~32K non-Python (~57% Python)** — note this differs from the seed-snippet ratio because "LLMs performing OSS-Instruct may produce code in a different programming language than the seed" ([ar5iv paper](https://ar5iv.labs.arxiv.org/html/2312.02120)).
- **Dedup/decontamination:** authors "removed identical samples or those sharing the same seed snippet"; applied "the same logic as StarCoder" to filter out problems containing docstrings/solutions from HumanEval and MBPP, docstrings from APPS, prompts from DS-1000, or questions from GSM8K — result: "the decontamination procedure only filters out 9 additional samples" (i.e. contamination against these benchmarks was already near-absent before the explicit filter) ([ar5iv paper](https://ar5iv.labs.arxiv.org/html/2312.02120)).
- **The Stack terms note:** the *seed* corpus (starcoderdata) is a filtered, permissively-licensed subset of **The Stack v1** (not v2) — see §2 below for The Stack v2's terms specifically, since OpenCodeInstruct (§1.4) uses v2 directly.

### 1.4 OpenCodeInstruct (seed anchor, NVIDIA, arXiv:2504.04030)

- **Size:** ~4.97M rows (paper/card state "5 million" samples); download size 6.86 GB, total (auto-converted Parquet) dataset size 19.07 GB ([nvidia/OpenCodeInstruct](https://huggingface.co/datasets/nvidia/OpenCodeInstruct)).
- **License:** **CC-BY-4.0** ([HF page](https://huggingface.co/datasets/nvidia/OpenCodeInstruct)).
- **Provenance:** fully synthetic. Uses the **Genetic-Instruct** framework (combining Self-Instruct + Evol-Instruct strategies); generation/response models: **Qwen2.5-32B-Instruct**, **Qwen2.5-Coder-32B-Instruct**, with **QwQ-32B-Preview** tested as an alternative solution-generator. Two seed collections: **1.43M generic instructions derived from Python functions in Stack V2**, and **25,443 algorithmic questions from the TACO dataset** (LeetCode/Codeforces/AtCoder etc.) ([arXiv HTML](https://arxiv.org/html/2504.04030v1)).
- **Language coverage:** **Python-only** — the paper states the 5M samples are all Python; no other language is included ([arXiv HTML](https://arxiv.org/html/2504.04030v1)).
- **Dedup/decontamination:** "Genetic-Instruct's inherent deduplication" via an n-gram-based approach, plus explicit **n-gram decontamination against evaluation benchmarks**; additional filtering removed samples with "noisy" Python snippets ([arXiv HTML](https://arxiv.org/html/2504.04030v1)).
- **The Stack v2 terms — resolved, see §2.**

### 1.5 KodCode-V1 (additional — largest fully test-verified set found)

- **Size:** 487,432 rows total (484K in `train` split + 3.34K flagged `use_with_caution`); total file size 2.64 GB; 12 subsets (Prefill 43K, Leetcode 27K, Codeforces 33K, Apps 21K, Taco 81K, Code Contests 36K, Algorithm 31K, Data Structure 34K, Docs 43K, Filter 77K, Package 7K, Evol 13K) ([KodCode/KodCode-V1](https://huggingface.co/datasets/KodCode/KodCode-V1)).
- **License: CC BY-NC-4.0 — non-commercial only** ([HF page](https://huggingface.co/datasets/KodCode/KodCode-V1)). This is the most restrictive license of any dataset in this dossier for commercial use; flag before including in any corpus whose downstream model might be released commercially.
- **Provenance:** fully synthetic; solutions and unit tests both generated by **`gpt-4o-0513`**, with up to 10 generation trials per question and self-verification via unit-test execution ([HF page](https://huggingface.co/datasets/KodCode/KodCode-V1)).
- **Language coverage:** Python-only.
- **Dedup/decontamination:** each row carries a `benchmark_similarity` field — "maximum cosine similarity between this question and all questions from HumanEval, MBPP, BigCodeBench, and LiveCodeBench" — allowing users to filter contaminated rows themselves, plus a `filter_reason` field for other quality flags ([HF page](https://huggingface.co/datasets/KodCode/KodCode-V1)). Note this is a *user-applied* filter mechanism, not a guarantee the shipped rows are already decontaminated.

### 1.6 StarCoder2-Instruct self-alignment set (additional — cleanest provenance)

- **Dataset:** `bigcode/self-oss-instruct-sc2-exec-filter-50k`.
- **Size:** 50,661 rows; 90.1 MB total file size ([HF page](https://huggingface.co/datasets/bigcode/self-oss-instruct-sc2-exec-filter-50k)).
- **License: ODC-BY** (Open Data Commons Attribution) ([HF page](https://huggingface.co/datasets/bigcode/self-oss-instruct-sc2-exec-filter-50k)).
- **Provenance — notably different from every other dataset here:** generated by **StarCoder2-15B itself**, with **no human annotation and no distillation from a larger/proprietary LLM** (no GPT-3.5/4, no Claude) ([HF page](https://huggingface.co/datasets/bigcode/self-oss-instruct-sc2-exec-filter-50k); background methodology in the SelfCodeAlign paper, arXiv:2410.24198, and the official announcement ([HF blog: StarCoder2-Instruct](https://huggingface.co/blog/sc2-instruct))). Pipeline: seed Python functions → machine-extracted "concepts" → synthetic instruction → response, filtered by **execution validation** (10 samples per instruction, kept only if they pass generated tests in a sandbox) ([HF blog](https://huggingface.co/blog/sc2-instruct)).
- **Language coverage:** Python-only (seed functions are Python).
- **Dedup:** the blog post states the SFT set is built "after deduplication" of the execution-validated pool, down to 50k instructions each paired with one randomly-selected passing response ([HF blog](https://huggingface.co/blog/sc2-instruct)); the underlying algorithm is not detailed further in the sources fetched this run (OPEN).
- **Contamination risk:** lowest of any dataset reviewed here — no proprietary-LLM output in the pipeline at all, sidesteps both OpenAI-ToS questions and reliance on a closed model's training-data leakage.

### 1.7 CommitPackFT — Python subset (additional — only human-authored source found)

- **Dataset:** `bigcode/commitpackft` (from the OctoPack project).
- **Size (total):** 702,062 rows across 277 languages, 1.58 GB; **Python subset: 56,025 rows, 132.68 MB** ([HF page](https://huggingface.co/datasets/bigcode/commitpackft)).
- **License:** dataset card is tagged **MIT**, but each row also carries the **original repository's own license** in a `license` field, since the content is real commit diffs pulled from GitHub repos — "all samples stem from permissively licensed repositories" ([HF page](https://huggingface.co/datasets/bigcode/commitpackft)). Practical implication: an MIT top-level tag does not override the fact that individual rows are governed by whatever permissive license their source repo used; a strict pipeline should retain/check the per-row `license` field rather than treating the whole set as blanket-MIT.
- **Provenance — human-authored, not model-generated:** real GitHub commit messages + before/after file diffs, filtered "to contain only high-quality commit messages that resemble natural language instructions" ([HF page](https://huggingface.co/datasets/bigcode/commitpackft)). This is the **only dataset in this dossier whose instruction text is human-written** rather than LLM-synthesized, making it valuable as a diversity/contamination-risk offset against the otherwise all-synthetic mix.
- **Dedup:** described as quality-filtered from the larger CommitPack corpus; specific dedup thresholds/near-duplicate-removal method not detailed in the source fetched this run (OPEN).

### 1.8 Tested-143k-Python-Alpaca (additional — execution-verified aggregate)

- **Dataset:** `Vezora/Tested-143k-Python-Alpaca`.
- **Size:** 143,327 rows ([HF page](https://huggingface.co/datasets/Vezora/Tested-143k-Python-Alpaca)).
- **License:** **Apache-2.0** ([HF page](https://huggingface.co/datasets/Vezora/Tested-143k-Python-Alpaca)).
- **Provenance:** derivative aggregate — Python code was extracted from the *output* fields of several existing Alpaca-format datasets (WizardLM's Evol datasets, "CodeUp's 19k", `sahil2801/CodeAlpaca`, Eric Hartford's Dolphin/dolphin-coder-sharegpt-290k, both Magicoder datasets, plus some hand-prompted GPT-4 questions), then **each code sample was actually executed**; kept only if it ran without error ([HF page](https://huggingface.co/datasets/Vezora/Tested-143k-Python-Alpaca)). **Important overlap note:** because this dataset is built *from* CodeAlpaca-20k, Evol-Instruct, and Magicoder, combining it with those raw sources (§1.1–1.3) in the same corpus mix double-counts substantially overlapping content — see recommended-mix reasoning in §3.
- **Language coverage:** Python-only (by construction — only Python code was extracted/tested).
- **Dedup:** creators state the result "was carefully deduplicated," but the specific method is not detailed in the source fetched this run (OPEN).
- **Contamination risk:** inherits whatever risk existed in its upstream sources (CodeAlpaca = `text-davinci-003`-generated, Magicoder = `gpt-3.5-turbo`-generated, Evol-Instruct = GPT-4-class-generated per WizardCoder methodology) — no independent benchmark-decontamination step is described beyond execution-testing (execution testing filters non-working code, not benchmark leakage).

### 1.9 Evol-Instruct-Code-80k-v1 / Evol-Instruct-Python-26k (additional — most restrictive license found)

- **Datasets:** `nickrosh/Evol-Instruct-Code-80k-v1` (parent, multi-language) and `mlabonne/Evol-Instruct-Python-26k` (Python-filtered child).
- **Size:** parent 78,264 rows ("78.3k"); Python-filtered child **26,588 rows** ([nickrosh page](https://huggingface.co/datasets/nickrosh/Evol-Instruct-Code-80k-v1); [mlabonne page](https://huggingface.co/datasets/mlabonne/Evol-Instruct-Python-26k)).
- **License:** parent dataset is tagged **CC-BY-NC-SA-4.0** ([nickrosh page](https://huggingface.co/datasets/nickrosh/Evol-Instruct-Code-80k-v1)) — the Python-filtered child's own card does not restate a license, so it should be treated as inheriting the parent's **non-commercial, share-alike** terms unless the child's authors state otherwise (not confirmed this run — OPEN). This is the **most restrictive license of any dataset reviewed here**: NC blocks commercial use (like KodCode), and SA additionally requires derivative *datasets* to be released under the same license — a real constraint if mixed into a corpus intended for permissive release.
- **Provenance:** "Open Source Implementation of Evol-Instruct-Code as described in the WizardCoder paper" (arXiv:2306.08568). **The specific generation model is not confirmed** — neither the dataset card nor the paper's abstract/arXiv listing page state which LLM ran the Evol-Instruct evolution+response step for this specific 80k set (fetched both; neither specifies) — OPEN item, do not assume GPT-3.5/4 without checking the full paper body.
- **Dedup/decontamination:** not described in the sources fetched this run (OPEN).

## 2. The Stack v2 license terms (explicit resolution, as required by brief)

Fetched directly from [`bigcode/the-stack-v2`](https://huggingface.co/datasets/bigcode/the-stack-v2):

- The Stack v2 does **not** apply one blanket license to the whole corpus. It **preserves each file's original repository license**. Only **permissively-licensed files and files with no declared license** are included — the card states: "To the best of our knowledge, all files contained in the dataset are licensed with one of the permissive licenses ... or no license," where "permissive" means approved by the **Blue Oak Council** or categorized **"Permissive"/"Public Domain" by ScanCode** (full per-file list in `license_stats.csv`).
- Explicit downstream obligation: **"Any use of all or part of the code gathered in The Stack v2 must abide by the terms of the original licenses, including attribution clauses when relevant."** This means a corpus derived from The Stack v2 (e.g. OpenCodeInstruct's 1.43M generic-instruction seed, §1.4) inherits a per-file attribution obligation from the original repos, not a single clean license grant.
- **Opt-out mechanism exists** — a "Am I In The Stack?" tool lets developers request removal; validated removal requests as of v2.1.0 (cutoff 2024-04-09) were honored.
- **Training-use condition:** users training models on bulk Stack v2 data must "adhere to the SoftwareHeritage principles for language model training," and bulk downloading requires agreeing to terms with Software Heritage and INRIA.
- **Not resolved by the fetched source:** whether an *instruction-tuning dataset derived from* Stack v2 content (rather than the raw code itself) can be freely redistributed/relicensed — the card's terms are written for the raw code corpus and don't explicitly address derived synthetic-instruction datasets. This matters directly for OpenCodeInstruct's CC-BY-4.0 self-declared license on data seeded from Stack v2 Python functions — flagged as **OPEN** (a legal-interpretation question, not a lookup we can close by searching further).

## 3. Recommended Python-only corpus mix + token estimate

**Reasoning.** Goal: maximize scale and license permissiveness while (a) avoiding heavy duplicate content across sources, (b) keeping at least one non-synthetic (human-authored) component to offset an all-LLM-generated corpus, and (c) flagging every license/provenance risk rather than silently absorbing it.

**Core mix (permissive-license branch — safe for broad including commercial-adjacent research use):**

| Dataset | Python rows | License | Provenance |
|---|---|---|---|
| OpenCodeInstruct | ~4.97M | CC-BY-4.0 | synthetic (Qwen2.5 family), n-gram decontaminated |
| Tested-143k-Python-Alpaca | 143,327 | Apache-2.0 | aggregated + execution-verified (supersedes raw CodeAlpaca-20k/Evol-Instruct/Magicoder — see below) |
| self-oss-instruct-sc2-exec-filter-50k | 50,661 | ODC-BY | self-generated by StarCoder2-15B only, no proprietary-LLM dependency |
| CommitPackFT (Python) | 56,025 | MIT + per-row original repo license | human-authored real commits |
| **Subtotal** | **≈5.22M rows** | | |

Excluded from the core mix, with reasons:
- **CodeAlpaca-20k / Evol-CodeAlpaca-V1 / Magicoder-OSS-Instruct-75K raw** — largely subsumed by Tested-143k-Python-Alpaca, which already extracted+execution-tested Python code from CodeAlpaca, Evol-Instruct, and Magicoder; including both would double-count near-duplicate content (§1.8). Also unresolved Python-only fractions for the first two (OPEN).
- **Evol-Instruct-Python-26k** — most restrictive license found (CC-BY-NC-SA-4.0, inherited); include only if the programme accepts a non-commercial, share-alike obligation on the resulting corpus.
- **KodCode-V1 (484K rows)** — held out of the "core" permissive mix solely because of its **CC BY-NC-4.0** non-commercial term; if this program's use is non-commercial research (consistent with the rest of this repo's academic framing), it is a strong *optional add* — it is the largest fully test-execution-verified set after OpenCodeInstruct, with per-row benchmark-similarity scores for user-side decontamination.

**Optional non-commercial-research add:** + KodCode-V1 (484K rows, CC BY-NC-4.0) → **≈5.71M rows total**.

**Token estimate (explicitly a rough, labeled estimate — no dataset here publishes an authoritative total token count):**

- The only directly sourced per-row token statistic in this dossier is Evol-CodeAlpaca-V1's median of 471 tokens/row (§1.2, not used in the core mix but useful as a conservative single-turn instruction+response anchor).
- Applying that as a **conservative floor** to the core mix's ≈5.22M rows: 5.22M × 471 ≈ **2.46B tokens**.
- OpenCodeInstruct rows are richer than a plain single-turn Alpaca sample (question + solution + unit tests + LLM judgement + execution status per §1.4), so the true count is likely higher. As a **cross-check upper bound**, OpenCodeInstruct's own stated 19.07 GB total (auto-converted Parquet) dataset size (§1.4), converted via a generic ~4-characters-per-token heuristic for English+code text (a standard rough approximation, not a dataset-specific measurement), gives 19.07×10⁹ ÷ 4 ≈ **4.8B tokens for OpenCodeInstruct alone** — noting this parquet-size figure likely includes non-prose fields (test/status columns) that inflate it, and may also reflect compressed rather than raw byte size, so this direction of error is uncertain.
- **Net estimate: order of magnitude ~2.5–5B tokens for the core mix**, with the wide range explicitly because per-dataset token counts are not published anywhere in the sources fetched this run. **Recommend the eventual data pipeline compute an exact count via the actual training tokenizer once selected** rather than relying on this estimate for capacity planning.

## 4. OPEN / unresolved items

1. **CodeAlpaca-20k license conflict** — HF mirror says CC-BY-4.0, original GitHub repo badge says Apache-2.0 (§1.1); not resolved to a single canonical answer this run.
2. **CodeAlpaca-20k and Evol-CodeAlpaca-V1 Python-only fractions** — neither dataset card states what fraction of rows are Python vs. other languages (§1.1, §1.2); would need a manual per-row language-field pass to resolve.
3. **Whether Stack-v2-derived instruction datasets can be freely relicensed/redistributed** — The Stack v2's terms (§2) are written for the raw code corpus, not for synthetic instruction data derived from it (e.g. OpenCodeInstruct's Stack-V2-seeded 1.43M subset); this is a legal-interpretation question, not resolvable by further search.
4. **`mlabonne/Evol-Instruct-Python-26k`'s own license** — its card states no license directly; treated as inheriting the CC-BY-NC-SA-4.0 of its parent `nickrosh/Evol-Instruct-Code-80k-v1`, but this inheritance is an assumption, not a confirmed statement (§1.9).
5. **Generation model for `nickrosh/Evol-Instruct-Code-80k-v1`** — neither the dataset card nor the WizardCoder paper's abstract/listing page (arXiv:2306.08568) name the specific LLM used to run Evol-Instruct for this dataset; would require reading the full paper body (not fetched this run) (§1.9).
6. **Exact dedup algorithms** for CommitPackFT (§1.7), Tested-143k-Python-Alpaca (§1.8), and StarCoder2's self-oss-instruct set (§1.6) — all three state deduplication was performed but none of the fetched sources specify the method (exact-match, n-gram, embedding-based, etc.).
7. **No authoritative total token count** exists for any dataset in this dossier except the derived Evol-CodeAlpaca-V1 estimate; the §3 mix's token estimate is a labeled rough range, not a measured fact.
8. **KodCode-V1's `use_with_caution` 3.34K rows** — excluded from this dossier's row counts where noted, but the exact reason/criteria for that flag beyond "predetermined filtering rules" was not detailed in the fetched source (§1.5).
